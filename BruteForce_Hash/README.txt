Постановка задачи.
Дана строка S длины n и строка Т длины m. В дальнейшем, не оговаривая каждый раз отдельно, будем предполагать,
что n>=m. Будем нумеровать символы строк начиная с 1.
Обозначим через S [i..j], 1 <= i < j <= n, подстроку строки S, начинающуюся в i-й позиции и заканчивающуюся в j-й позиции.
Требуется найти все вхождения строки Т в строку S, т. е. указать все позиции строки S,
начиная с которых читается строка Т. Множество вхождений, которое надо построить, формально описывается так:
М = {i|1 <= i <= n — m + 1, S[i..i+m] = Т}.
Например, пусть S = abcabdecab, Т = cab, тогда М = {3; 8}.
Или такой пример: пусть S = abababacaba, Т = aba, тогда
М = {1; 3; 5; 9}. Таким образом, мы допускаем «перекрытие» — ситуацию, когда расстояние между двумя соседними
вхождениями меньше длины строки Т.

формат запуска скрипта:
node bruteforce.js     inp.txt            aba
где:                 входной файл    искомая строка

или

node hash.js     tom1.txt          "князь Андрей"
где:           входной файл        искомая строка

В директории находится файл report.txt с отчетом о работе на 3 разных тестах.
